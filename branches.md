 The standard workflow for switching branches cleanly:

  # 1. Check current status
  git status

  # 2. Stage and commit your changes on current branch
  git add .
  git commit -m "Your commit message"

  # 3. Switch to another branch
  git checkout <branch-name>

  If you have uncommitted changes and need to switch:

  # Option A: Stash (temporary save)
  git stash
  git checkout <other-branch>
  # Later, come back and restore:
  git checkout <original-branch>
  git stash pop

  # Option B: Commit work-in-progress
  git add .
  git commit -m "WIP: description"
  git checkout <other-branch>

  For your workflow (feature branches):

  # Working on feature branch
  git checkout 26-dep-001-docker-compose-stack

  # Do work...

  # Save and commit
  git add .
  git commit -m "feat(deploy): add docker-compose stack"

  # Push to remote
  git push origin 26-dep-001-docker-compose-stack

  # Switch to another feature
  git checkout 5-sdk-001-async-logger-core

  Key rule: Always commit or stash before switching branches to avoid conflicts.
